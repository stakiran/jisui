# jisui

## 解説

### 自炊とは
ここでは手元で閲覧している資料の一部をキャプチャし、個人用の検索・メモ補助としてテキスト化することを指します。

### 利用上の注意

- 利用規約で禁止されているサービス・コンテンツには使用しないでください
- 技術的保護手段の回避を伴う利用はしないでください
- 適法性の判断は利用者自身の責任で行ってください

### 使い方
jisui では 2 つのスクリプトを使います:

- 1: capture_tool.py を使って「指定した矩形範囲内」を「指定したホットキー」でキャプチャできるようにします
- 2: ocr_agent.py を使って、1 が出力した画像を文字起こしします
    - Claude API を使います
    - 文字起こしは text.md に、番号ごとに見出しをつくって保存します

最低限これだけでも運用できますが、キャプチャを手作業で行うのがだるいでしょう。

- 3: auto_capture.py を使って、1: を準備できた後の操作を自動化できます
    - w 秒待った後に開始し、n 秒ごとに指定ホットキーを押す処理を count 回繰り返します

## 使い方

### ターミナル1: キャプチャツール
```bash
python capture_tool.py --output-dir ./captures --hotkey f12
```

### ターミナル2: 文字起こしエージェント
```bash
python ocr_agent.py --watch-dir ./captures --output text.md
```

### 動作の流れ

1. **F12** を押すと画面が暗転し、矩形選択モードになる
2. マウスでドラッグして範囲を選択
3. `captures/1.png` として保存される（次回は `2.png`, `3.png`, ...）
4. エージェントがファイルを検知し、Claude API で文字起こし
5. `text.md` に以下の形式で追記される:

```markdown
# 1

（1.png の文字起こし結果）

# 2

（2.png の文字起こし結果）
```

## オプション

| 引数 | キャプチャ | エージェント | 説明 |
|------|-----------|-------------|------|
| `--output-dir` / `-o` | ✅ | — | キャプチャ保存先 |
| `--hotkey` / `-k` | ✅ | — | 起動キー (f1〜f12) |
| `--watch-dir` / `-w` | — | ✅ | 監視ディレクトリ |
| `--output` / `-o` | — | ✅ | 出力 Markdown |
| `--model` / `-m` | — | ✅ | Claude モデル名 |

## 注意事項

- エージェント起動時に、未処理の既存ファイルも自動的に処理します
- `text.md` に既に `# (番号)` がある場合はスキップします（重複防止）
- コスト重視なら `--model claude-haiku-4-5-20251001` を指定してください
